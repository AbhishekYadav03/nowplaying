# 🎵 NowPlaying

Real-time media sharing for friends and couples. Built with Flutter 3.41.2 + Firebase.

## Prerequisites

| Tool             | Version       |
|------------------|---------------|
| Flutter SDK      | 3.41.2 stable |
| Dart SDK         | 3.11          |
| Android Studio   | Hedgehog+     |
| Xcode            | 16+ (macOS)   |
| CocoaPods        | latest        |
| Firebase CLI     | 15.1.0+       |
| FlutterFire CLI  | latest        |
| Java             | 17+           |
| Node.js          | 18+           |

## 1. Clone & Install

```bash
git clone https://github.com/your-org/nowplaying.git
cd nowplaying
flutter pub get
```

## 2. Firebase Setup

```bash
# Install Firebase CLI
npm install -g firebase-tools
firebase login

# Install FlutterFire CLI
dart pub global activate flutterfire_cli

# Configure Firebase (creates lib/firebase_options.dart automatically)
flutterfire configure --project=heart-beat-1108
```

In Firebase Console, enable:
- **Authentication** → Email/Password + Google
- **Firestore** → Production mode
- **Cloud Messaging** → (auto-enabled)
- **Cloud Functions** → Requires Blaze plan

## 3. Deploy Firestore Rules

```bash
firebase deploy --only firestore:rules
```

## 4. Deploy Cloud Functions

```bash
cd functions
npm install
cd ..
firebase deploy --only functions
```

## 5. Android Setup

1. Download `google-services.json` from Firebase Console
2. Place in `android/app/google-services.json`
3. The user must grant **Notification Listener** permission on first launch
   (Settings → Special App Access → Notification Access → NowPlaying ✓)

## 6. iOS Setup

1. Download `GoogleService-Info.plist` from Firebase Console
2. Drag into `ios/Runner/` in Xcode
3. In Xcode, add capabilities:
   - **Push Notifications**
   - **Background Modes → Background fetch + Remote notifications**

> ⚠️ **iOS media detection** from third-party apps like Spotify is restricted.
> Use the Manual Broadcast feature in Profile screen as a fallback.
> For Spotify: integrate the [Spotify iOS SDK](https://developer.spotify.com/documentation/ios) separately.

## 7. Run

```bash
flutter run
```

## Project Structure

```
lib/
├── main.dart               # Entry point
├── firebase_options.dart   # Auto-generated by FlutterFire CLI
├── app/
│   ├── app.dart            # MaterialApp + GoRouter
│   ├── theme.dart          # AppColors, AppTheme
│   └── router.dart         # GoRouter with auth redirect
├── features/
│   ├── auth/               # Login / signup
│   ├── feed/               # Now playing friend feed
│   ├── friends/            # Add / manage friends
│   ├── media/              # Media detection (platform channel)
│   └── profile/            # Profile, sharing toggle
├── models/                 # UserModel, NowPlayingModel, ReactionModel
├── services/               # FirestoreService, AuthService, MediaService, FCM
└── widgets/                # NowPlayingCard, GradientButton, MainShell

android/
├── MediaPlugin.kt          # MediaSession EventChannel
├── MediaNotificationListenerService.kt
└── MainActivity.kt

functions/src/
└── index.ts                # Cloud Functions (reactions FCM, cleanup)
```

## Key Commands

```bash
# Run app
flutter run

# Build release
flutter build apk --release        # Android
flutter build ios --release         # iOS

# Code generation (Riverpod)
dart run build_runner build --delete-conflicting-outputs

# Run tests
flutter test

# Deploy Firebase
firebase deploy --only functions
firebase deploy --only firestore:rules
```

## Architecture

- **State**: Riverpod 2.x with code generation
- **Navigation**: GoRouter 14.x with auth redirect
- **Backend**: Firebase (Firestore realtime + FCM push)
- **Media**: Android MediaSession via EventChannel; iOS manual fallback
